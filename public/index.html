API Reference
POST/api/v1/workRequest a chunk of keyspace to scan
Request
{
  "name":     "rig-01",   // worker identifier (any unique string)
  "hashrate": 5000000     // keys/s, used to size the chunk
}
Response
{
  "job_id":    42,         // echo back in submit / heartbeat
  "start_key": "0x...00", // inclusive range start (hex)
  "end_key":   "0x...1c"  // inclusive range end   (hex)
}
POST/api/v1/submitReport a completed or solved chunk
Request
{
  "name":    "rig-01", // must match name used in /work
  "job_id":  42,       // job_id returned by /work
  "status":  "done",   // "done" | "found"
  "found_address": ""  // only when status == "found"
}
Response
{
  "accepted": true  // false if job_id unknown / already closed
}
POST/api/v1/heartbeatKeep worker alive during long jobs
Request
{
  "name":   "rig-01", // worker identifier
  "job_id": 42        // resets 15-min reclaim timer
}
Response
{
  "ok": true  // call every ~120s while working
}
